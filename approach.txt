1. ROOT CAUSE ANALYSIS  (why v1 scored ~86 despite OOF ~98)
────────────────────────────────────────────────────────────────────────
v1 used random KFold on spatial-temporal data. This leaks future
timestamps and nearby geohash demand values into validation folds,
making OOF scores artificially inflated (~98) while the true
leaderboard generalization was far lower (~86).

FIX: Switched to GroupKFold keyed on geohash — each fold validates on
entirely different spatial locations, giving an honest estimate of
out-of-sample performance.

3. NEURAL NETWORK ARCHITECTURE
────────────────────────────────────────────────────────────────────────
  Architecture:   Residual MLP
  Input Proj:     Linear(n_features → 256) → BatchNorm → GELU
  Residual Blocks (×4):
                  Linear(256→256) → BatchNorm → GELU → Dropout(0.15)
                  → Linear(256→256) → BatchNorm → skip connection
  Output Head:    Linear(256→64) → GELU → Linear(64→1) → Sigmoid
  Loss:           HuberLoss(delta=0.1)  — robust to outlier spikes
  Optimizer:      AdamW(lr=1e-3, weight_decay=1e-4)
  Scheduler:      CosineAnnealingLR(T_max=50)
  Early Stopping: patience=15 on validation MSE
  Preprocessing:  StandardScaler on all features (NaN → 0)


────────────────────────────────────────────────────────────────────────
4. FEATURE ENGINEERING  (73 total features)
────────────────────────────────────────────────────────────────────────
a) Geohash Spatial Resolution
   - Custom base-32 decoder → lat/lon for every geohash
   - 10 unseen test geohashes mapped to nearest training neighbor
     (Euclidean distance on decoded coordinates)
   - Precomputed 3 nearest spatial neighbors per geohash for spillover

b) Spatial Clustering
   - KMeans(k=20) on lat/lon of all geohashes
   - cluster_ts_mean: mean demand in same cluster at same timestamp
   - lag0_vs_cluster: deviation of cell demand from cluster average

c) Historical Lag Features (Day 48 & early Day 49 pivot tables)
   - Exact lags: 0, ±15, ±30, ±60, ±90, ±120 minutes
   - Spatial spillover: mean demand of 3 nearest neighbors at lag 0
   - Aggregates per lag set: mean, max, std, median, min, range

d) Day-over-Day Trend
   - demand_trend = d49e_lag0 − d48_lag0    (momentum)
   - demand_ratio = d49e_lag0 / d48_lag0    (relative change)

e) Cyclical Temporal Encodings
   - hour_sin, hour_cos  (period = 24 h)
   - min_sin,  min_cos   (period = 1440 min)
   - Time-of-day flags: is_rush, is_morning, is_afternoon, is_night

f) Road & Environment Features
   - RoadType encoded: Residential=0, Street=1, Highway=2
   - LargeVehicles_enc, Landmarks_enc, Weather_enc
   - Temperature (missing → group mean), temp_extreme flag
   - temp_weather_interaction = Temperature × Weather_enc

g) Geohash Statistics (from Day 48)
   - gh_mean48, gh_std48, gh_median48, gh_max48, gh_min48
   - ts_mean48, ts_std48  (time-slot level averages)
   - gh_ts_deviation = d48_lag0 − ts_mean48

h) Interaction Features
   - lag0_x_ts    = d48_lag0 × ts_mean48
   - lanes_highway = NumberofLanes × (RoadType == Highway)
   - pct_of_day   = time_mins / 1440
